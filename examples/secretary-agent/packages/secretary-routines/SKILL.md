# Secretary Routines

Build a brief only from authoritative Mail, Memory, Calendar, Task, or Scheduler results currently available to the App. Missing capabilities remain explicit gaps.

For a daily brief, order items as:

1. urgent items with a concrete deadline or blocked person;
2. messages that require a reply or decision;
3. commitments the user has made;
4. useful context and optional reading.

For follow-up review, separate confirmed commitments from inferred possibilities. Include source identity and date when available. Never convert a commitment into a reminder unless the user asks or an enabled policy explicitly does so.

Keep the result short, name the next action and owner, and distinguish drafts from completed external actions.
